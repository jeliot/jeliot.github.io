Slowdown 1: Heavy Computation The expression f_hard(a) = sum(n=1 to 10^k, integral(0 to n, sin(t−a)dt)) combines a summation with a nested definite integral. When k=4, this evaluates 10,000 integrals per frame. The function is called as f_hard(v+A.x), making it dependent on both the slider v and the movable point A. This is by far the most computationally expensive single expression in the graph.
Slowdown 2: Plotting Overload The parametric expression (cos(t), sin(t)) · repeat(1, 1000) creates 1,000 overlapping unit circles. Slowdown 2b: Implicit Curve Multiplication The expression y = [-8,-7.8...8] + cos(x) generates 81 separate implicit curves (one for each value in the list range). Combined with y = sin(x) + x, Desmos must find all intersection points between 81 curves and the sine function, creating massive rendering and root-finding load.

⚑ Red Herring: Very Long List The list comprehension [A.x + v + xy for x=[0…32], y=[0…32]] references the movable point A and slider v, so it does recalculate on input changes. However, as a single list comprehension generating 1,024 values, it’s far cheaper than 1,024 separate expressions. Estimated cost: ~9.3 vs 1,024.0.
